Dockerfile 指令

1.CMD

CMD指令用于指定一个容器启动时要运行的命令。类似于Run指令，而CMD指容器被启动时执行的指令。[Dockerfile中只能有一条CMD]

2.ENTRYPOINT

ENTRYPOINT指令与CMD指令非常类似，ENTRYPONT指令不容易在容器启动时被覆盖。实际上 docker run 命令行中的任何参数都会被当作参数再次传递给ENTRYPOINT中的指令。

3.WORKDIR

WORKDIR  指令用来在从镜像创建一个新容器时，在容器内部设置一个工作目录，ENTRYPOINT或CMD指定的程序会在这个目录执行。

4.ENV

ENV 指令用来在镜像构建过程中设置环境变量。也可以使用 docker run 命令行的-e标志来传递环境变量。[运行时有效]

example:

5.USER

USER指令用来指定该镜像会以什么样的用户去运行。默认为 root

6.VOLUME

VOLUME 指令用来向基于镜像的容器添加卷。一个卷是可以存在于一个或者多个容器内的特定的目录，这个目录可以绕过联合文件系统，并提供如下共享数据或者对数据进行持久化的功能。
  • 卷可以在容器间共享和重用
  • 一个容器可以不是必须和其他容器共享卷
  • 对卷的修改是立即生效的
  • 对卷的修改不会对更新镜像产生影响
  • 卷会一直存在直到没有容器再使用它
  • docker cp 允许从容器复制文件以及复制文件到容器

example

7.ADD

ADD 指令用来将构建环境下的文件和目录复制到镜像中。比如，在安装一个应用时。ADD指令需要源文件位置和目的文件位置两个参数。

example

8.COPY

COPY 指令非常类似于 ADD，它们的根本区别是COPY只关心在构建上下文中复制本地文件，而不会去做文件提取和解压的工作.[copy只能复制与Dockdrfile同一目录下的文件]

example

9.LABEL

LABEL 指令用于为 Docker镜像添加元数据，元数据以键值对的形式出现。

example

10.STOPSIGANAL

STOPSIGANL 指令用来设置停止容器时发送什么系统调用信号给容器。

11.ARG

ARG指令用来定义可以在 docker build 命令运行时传递给构建运行时的变量，我们只需要在构建时使用--build-arg 标志即可。

example

12.ONBUILD

ONBUILD 指令能为镜像添加触发器。当一个镜像被用做其他镜像的基础镜像时，该镜像中的触发器将会被执行。
